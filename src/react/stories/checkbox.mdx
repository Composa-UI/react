# Checkbox

A checkbox toggles a single option **on or off** — used for binary choices like settings
in the design editor, and for selecting individual rows in the admin interface.

<DocTabs>

<DocTab label="Usage">

<Canvas of={CheckboxStories.Playground} />

### Overview

#### When to use

- **Checkbox** — for opt-in settings, multi-select lists, "select all" rows, and any
  choice that applies on a later Save/submit rather than immediately.
- **Switch instead** — _if this state changes immediately, you may want to consider using
  a switch instead._
- **Radio instead** — when the user picks one option from a mutually exclusive set.

#### Variants

- **On / Off** — the two base states.
- **Mixed** — an indeterminate state, used when a checkbox has nested checkboxes with a
  mix of "on" and "off" states.

### Anatomy

<Canvas of={CheckboxStories.Anatomy} />

- **Box** — the 16 × 16 control that carries the on / off / mixed glyph.
- **Label** — usually present; names the option. _If a checkbox is disabled, the label is
  disabled as well._
- **Description** — optional text aligned under the label, for cases that need more room
  to add context to an option (it may briefly highlight side effects or conditions of the
  setting).

### Content

- **Label** — checkboxes are usually accompanied by a label naming the option.
- **Description** — optional supporting text aligned with the label, e.g. _"Helpful
  description of the setting which may briefly highlight side effects or conditions of the
  setting."_
- **Label rules** _(proposed)_ — keep labels short and in sentence case (only the first
  word and any proper nouns capitalized). Never truncate a checkbox label with an
  ellipsis — let it wrap instead. A programmatic label is always required, even when
  visually hidden.

#### Do / Don't _(proposed)_

- **Do** let a long label wrap beneath the box so the box and the label's first line stay
  top-aligned.
- **Don't** truncate a label with an ellipsis or vertically center wrapped text against
  the 16 × 16 box.

_Rows/notes marked (proposed) are inferred, Carbon-benchmarked additions awaiting confirmation._

### Behaviors

A checkbox combines a **type** (On / Off, with Mixed via the mixed glyph) with a **state**
(Default / Focused) and the `muted` (off) and `disabled` flags. Focus shows a 2px inset
ring; disabling the checkbox also disables its label.

#### States

| Type | State | Box background | Border | Glyph |
|---|---|---|---|---|
| On | Default | `color.bg.brand` | `color.border.selected-strong` | `icon.16.check` |
| On | Focused | `color.bg.brand` + 2px inset white ring | `color.border.selected-strong` | `icon.16.check` |
| On | Disabled | `color.bg.disabled` | none | `icon.16.check` (disabled fill) |
| Mixed | Default | `color.bg.brand` | `color.border.selected-strong` | `icon.16.mixed` |
| Mixed | Focused | `color.bg.brand` + 2px inset white ring | `color.border.selected-strong` | `icon.16.mixed` |
| Mixed | Disabled | `color.bg.disabled` | none | `icon.16.mixed` (disabled fill) |
| Off | Default (muted) | `color.bg.secondary` | `color.border` | none |
| Off | Focused (muted) | `color.bg.secondary` | `color.border.selected-strong` | none |
| Off | Disabled | none | `color.border.disabled` | none |
| Off / On | Read-only _(proposed)_ | inherits state | `color.border.disabled` | glyph at full `icon.16.check` fill; label stays `color.text`, not in tab order |
| Off / On | Error _(proposed)_ | inherits state | `color.border.error` _(proposed)_ | inline message below in `color.text.error` _(proposed)_; label stays `color.text` |

#### Screen reader & labeling

- The checkbox exposes a `checkbox` role with `aria-checked` (`true` / `false`, or `mixed`
  for the indeterminate state).
- **`label` is the accessible name** — always set a meaningful one, or the checkbox is
  unlabeled to assistive tech.
- A disabled checkbox disables its label too, so the whole control reads as inactive.
- A `description`, when present, gives extra context aligned under the label.
- **Checkbox-group semantics** _(proposed)_ — wrap a set of related checkboxes in a
  `<fieldset>` with a `<legend>` so screen readers detect the set and announce the group
  label as its accessible name; individual labels remain each checkbox's own name. A group
  carries its own label plus helper or error text, and read-only / disabled / error states
  can apply at the group level.

#### Focus

- The focused checkbox shows a 2px inset ring meeting the visible-focus requirement
  (`color.border.selected-strong`).
