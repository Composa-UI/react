# Cursors

The cursor tells the user what the pointer will do where it sits — selecting, drawing, resizing, panning, or chatting — and the system swaps it as the context and active tool change.

### Overview

Cursors are a system-level visual foundation: a 32 × 32 glyph that follows the pointer and changes shape to signal what an action will do at the current spot. The set spans four families — the main editing cursors, multiplayer (presence) cursors, system (macOS) cursors, and chat cursors that carry a speech bubble.

Each cursor is drawn to a shared construction guide: a base pointer vector with a hotspot ellipse at (28, 28) and diagonal guide lines at 45° / 135°, so every glyph aligns to the same active point.

### When each cursor appears

The shape reflects the active tool and what lies under the pointer:

- **Default pointer** — `cursor-black` for general pointing; `cursor-click` over an interactive target; `cursor-not-allowed` where the action is blocked.
- **Move / nudge** — `cursor-move`, `cursor-black-move`, and the constrained `cursor-black-move-h` / `cursor-black-move-v` while dragging an object.
- **Resize** — `cursor-resize`, `cursor-resize-scale`, `cursor-resize-padding`, and `cursor-resize-padding-bounded` over selection or padding handles.
- **Rotate** — `cursor-rotate` just outside a selection corner.
- **Path & vector editing** — `cursor-pen`, `cursor-pen-insert`, `cursor-pen-remove`, `cursor-pen-snap`, `cursor-bend`, `cursor-edit-corners`, `cursor-edit-radius`, `cursor-edit-ratio`, `cursor-convert`, `cursor-break`, `cursor-break-move`, `cursor-break-snap`.
- **Drawing & painting** — `cursor-pencil`, `cursor-marker`, `cursor-highlighter`, `cursor-eraser`, `cursor-dropper`, `cursor-paint-bucket` (and `-insert` / `-remove`), `cursor-stamp`, `cursor-wand`, `cursor-magic-handle`, `cursor-washi-tape`.
- **Frames & layout** — `cursor-frame`, `cursor-frame-snap`, `cursor-origin`, `cursor-position-absolutely`, `cursor-deep-parent`, `cursor-snap`, `cursor-duplicate`, `cursor-duplicate-snap`.
- **Text** — `cursor-text`, `cursor-text-curve`, and the missing-font variants (`cursor-missing-font`, `-light`, `-dark`).
- **Navigation** — `cursor-hand-new` / `cursor-hand-press` for panning, `cursor-zoom-in` / `cursor-zoom-out`, `cursor-offscreen`.
- **Selection & annotation** — `cursor-lasso`, `cursor-crosshair`, `cursor-comments-pin`, `cursor-comments-next-pin`, `cursor-annotate`, `cursor-guide-delete`, `cursor-prototyping`, `cursor-pizza`.
- **Multiplayer** — a presence cursor in one of seven colors (`cursor-multiplayer-blue` / `-yellow` / `-purple` / `-pink` / `-red` / `-green` / `-gray`) marks each other person's pointer.
- **System (macOS)** — when the OS owns the pointer, the native set shows: `cursor-system-arrow`, `-add`, `-not-allowed`, `-poof`, `-busy-but-clickable`, `-context-menu`, `-zoom`, `-crosshair`, `-text`, `-hand-pointing`, `-hand-open`, `-hand-closed`, `-resize`, `-move`.
- **Chat** — typing a message attaches a colored speech bubble to the pointer (`cursor-chat-*`).

### Anatomy

- **Pointer glyph** — the cursor symbol itself, drawn at 32 × 32. Its **hotspot** — the active point the OS reads — sits at (28, 28) on the construction guide, marked by an 8px ellipse with a 14px union and a 13px inner ellipse at the tip.
- **Construction guide** — a redline diagram (`_cursor-guide`) shows the base vector, two 90.51px diagonal guides at 45° and 135°, the hotspot, and the bounding 15px square the pointer is built within, so new cursors stay aligned to the same hotspot.
- **Chat bubble** — chat cursors compose the base `cursor-black` pointer (a 14px flat arrow glyph) with a speech bubble offset to (left 16, top 16). The bubble is a tail-cornered pill (radius 2 / 99 / 99 / 19 tl/tr/br/bl) holding a short text label.

### Behaviors

- The cursor **changes shape with the active tool and target** — pointing, then click, then move/resize/rotate as the pointer enters a handle or interactive region.
- Where an action is **not permitted**, the pointer shows `cursor-not-allowed` (or the system `cursor-system-not-allowed`).
- **Snapping** variants (`-snap`, `cursor-frame-snap`, `cursor-pen-snap`, `cursor-duplicate-snap`, `cursor-break-snap`) appear while a drag is aligning to a guide or edge.
- A **chat cursor** carries a live label the user is typing; its default text reads _"Say something"_ until replaced.
- When the **OS owns the pointer** (drag-and-drop, busy states, native menus), the system (macOS) family is shown instead of the editing set.

```css
/* Apply a cursor to a surface. Custom cursors set the image plus the hotspot
   x/y (28 28) so the active point lands on the pointer tip; a system keyword
   is the fallback when the image fails to load. */
.canvas {
  cursor: url("/cursors/cursor-black.svg") 28 28, default;
}

.canvas--moving {
  cursor: url("/cursors/cursor-move.svg") 28 28, move;
}

.canvas--blocked {
  cursor: not-allowed;
}
```

#### Pointer & keyboard

- Cursors are a **pointer affordance only** — they communicate intent to mouse/trackpad users and never replace a keyboard path. Every action signaled by a cursor (move, resize, select, pan) must also be reachable from the keyboard.
- A custom cursor always declares a **system-keyword fallback** (`default`, `move`, `text`, `not-allowed`, …) so the pointer stays meaningful if the custom image fails to load.
- The **hotspot** (28, 28) must match the visual tip, so the point the user aims with is the point the OS acts on.

#### Screen reader

- Cursor shape is **not exposed to assistive tech** and carries no accessible name. State that a cursor implies — disabled, busy, draggable — must be conveyed independently (e.g. `aria-disabled`, `aria-busy`, a live region, or visible text), never by the cursor alone.

#### Focus & reduced motion

- The cursor is not a focus indicator; focusable surfaces still need a visible focus ring of their own.
- Cursors are static glyphs (no animation), so there is no reduced-motion concern — except the system busy/spinner cursor, which is owned by the OS and honors the platform's motion settings.
