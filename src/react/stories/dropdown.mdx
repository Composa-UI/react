# Dropdown

A dropdown works like a select menu: it lets you **choose one option from a menu**. It is
styled as an input, not a button — a surface with a 1px border that stays put on
interaction rather than filling.

<Canvas of={DropdownStories.Playground} />

### When to use

- **Dropdown** — to pick a single value from a known set of options, especially in a
  property panel or form row.
- **Menu instead** — for richer surfaces that need grouping, search, or actions rather
  than a single selected value, compose a Menu separately.

### Variants

- **Icon-leading** — a leading glyph sits at the left of the row, then the value, then the
  trailing chevron.
- **Icon-less** — no leading glyph; the value starts after a small left padding
  (`spacer-2`), then the trailing chevron. A trailing chevron is always present.

### Anatomy

- **Surface** — a bordered row, 24px high, with `radius-medium` corners and a `color.bg`
  surface that does not fill on interaction.
- **Leading icon** (optional) — a 24px icon box at the left.
- **Value** — the selected option text, filling the row.
- **Chevron** — a trailing `chevron.down` glyph in a 24px box, always present, signaling
  the menu can open.

### Content

- The value is the selected option, rendered as plain text (for example "Value" or
  "Longer value" on the specimens).
- Like the Value component, option menus **do not show a border by default** — borders are
  intended to appear on hover and focus.
- **Label** _(proposed)_ — keep the label short and concise, limited to a single line; do not
  drop the label in favor of placeholder text.
- **Placeholder** _(proposed)_ — placeholder text is optional and signals interaction (for
  example "Choose an option"); it disappears on selection, so never put critical information
  there.
- **Helper text** _(proposed)_ — optional helper text sits beneath the control for selection
  guidance and persists while the control is focused.

### Behaviors

#### Width

- **Flexible width (default)** — a dropdown adjusts its width to its text length, hugging
  its content. A longer value produces a wider control.
- **Snap to grid** — to keep property panels consistent, dropdowns can snap to the flexible
  grid, locking to one, two, or three columns:

  | Grid lock | Width |
  |---|---|
  | One column | 88 |
  | Two columns | 184 |
  | Three columns | 184 |

#### States

| State | Border | Label | Surface |
|---|---|---|---|
| Default | `color.border` | `color.text` | `color.bg` |
| Hover | `color.border` | `color.text` | `color.bg` |
| Focus | `color.border.selected` | `color.text` | `color.bg` |
| Disabled | `color.border` | `color.text.tertiary` | `color.bg` |
| Read-only _(proposed)_ | `color.border` | `color.text` | `color.bg` |
| Error _(proposed)_ | `color.border.error` _(proposed)_ | `color.text` | `color.bg` |
| Warning _(proposed)_ | `color.border.warning` _(proposed)_ | `color.text` | `color.bg` |

_Rows/notes marked (proposed) are inferred, Carbon-benchmarked additions awaiting confirmation._

- **Default / Hover** — the surface stays white with its border; the control does not fill.
- **Focus** — the border switches to the selection-blue `color.border.selected`.
- **Disabled** — the value dims to `color.text.tertiary`; the control reads as inactive.
- **Read-only** _(proposed)_ — the value stays full-strength `color.text` (not dimmed like
  disabled) and the control is not focusable and offers no chevron interactivity; it reads as
  "value shown, not editable" rather than inactive.
- **Error** _(proposed)_ — the border uses `color.border.error` _(proposed)_ with an optional
  trailing error glyph; pair with helper/error text below the control.
- **Warning** _(proposed)_ — the border uses `color.border.warning` _(proposed)_; pair with
  helper text below the control.

#### Keyboard

| Key | Result |
|---|---|
| `Tab` | Moves focus to the dropdown |
| `Space` / `Enter` | Opens the menu |
| `Esc` | Closes the menu without changing the value |
| `Enter` / `Space` / `Down` _(proposed)_ | Opens the menu and focuses the first or currently selected option |
| `Up` / `Down` _(proposed)_ | Moves through the options while the menu is open |
| `Enter` _(proposed)_ | Selects the focused option |
| `Esc` _(proposed)_ | Closes the menu without changing the value |
| Typeahead _(proposed)_ | Typing a character jumps to the matching option |

_Rows/notes marked (proposed) are inferred, Carbon-benchmarked additions awaiting confirmation._

#### Screen reader & labeling

- The dropdown exposes its current value as the accessible name, so set a meaningful
  `label` — otherwise the control is unlabeled to assistive tech.
- The trailing chevron is decorative and is not announced on its own.
- **Listbox / option roles** _(proposed)_ — the trigger is a `button` with
  `aria-haspopup="listbox"` and an `aria-expanded` that reflects the open state; the menu is a
  `listbox` of `option`s, and the selected option is conveyed to assistive tech.

#### Focus

- The focused dropdown shows the selection-blue border (`color.border.selected`), meeting
  the visible-focus requirement.
