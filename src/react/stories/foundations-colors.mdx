# Colors

Composa exposes a semantic-role color layer. Components consume `--composa-color-*` tokens. They never read raw primitives. Each semantic token resolves through a `--figma-composa-*` primitive.

The model is mode-driven. The same token name resolves to a different value in light and dark. Light is the default. Dark applies under `[data-composa-mode="dark"]`. Each swatch below shows both values in equal-width chips: the light value in a light container, the dark value in a dark container, with the resolved value under each. When a token has no dark override, its dark value equals its light value.

Values are shown in a single consistent format. Solid colors read as hex; tokens that carry opacity (Figma authors them as `rgba`) read as the same hex with a trailing percentage, so `rgba(0,0,0,.5)` shows as `#000000 50%`.

export const formatValue = (value) => {
  const match = value.match(/rgba?\((\d+),\s*(\d+),\s*(\d+)(?:,\s*([\d.]+))?\)/);
  if (!match) return value;
  const hex = "#" + [match[1], match[2], match[3]].map((n) => Number(n).toString(16).padStart(2, "0")).join("");
  const alpha = match[4] === undefined ? 1 : parseFloat(match[4]);
  return alpha < 1 ? `${hex} ${Math.round(alpha * 100)}%` : hex;
};

export const ChipBox = ({ token, value, mode }) => (
  <div data-composa-mode={mode === "dark" ? "dark" : undefined} style={{ width: 84, flexShrink: 0, boxSizing: "border-box", textAlign: "center", background: mode === "dark" ? "#1e1e1e" : "#ededed", border: `1px solid ${mode === "dark" ? "#4d4d4d" : "#d9d9d9"}`, borderRadius: 8, padding: 6 }}>
    <div style={{ width: 36, height: 36, margin: "0 auto", borderRadius: 6, background: `var(${token})`, border: `1px solid ${mode === "dark" ? "rgba(255,255,255,0.18)" : "rgba(0,0,0,0.15)"}` }} />
    <div style={{ fontSize: 10, opacity: 0.7, marginTop: 4, color: mode === "dark" ? "#fff" : "#1a1a1a" }}>{mode === "dark" ? "Dark" : "Light"}</div>
    <div style={{ fontSize: 9, fontFamily: "monospace", marginTop: 1, whiteSpace: "nowrap", color: mode === "dark" ? "#fff" : "#1a1a1a" }}>{formatValue(value)}</div>
  </div>
);

Two distinct hues carry different jobs. Each resolves in both modes.

Brand is the orange product identity. Accent is the blue used for selection states, focus rings, and the active selection surface. They are not interchangeable.

## Modes

Composa curates 2 color modes. Light is the default. Dark applies under `[data-composa-mode="dark"]`. Every token resolves in both modes. A token with no dark override keeps its light value in dark.
