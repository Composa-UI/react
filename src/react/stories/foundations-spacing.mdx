# Spacing

Spacing is a base-4 ramp. Each step is a multiple of 4px. Use these tokens for gaps, padding, and margins. The bars below render at the live token width.

The ramp runs 0, 4, 8, 12, 16, 24, 32, 40. The 12px step is named `--composa-space-2-5` because it sits between 8 and 16. The remaining steps are whole indices.

## Base-4 ramp

The scale is a strict base-4 ladder. Every value is `4 × n`. This keeps vertical and horizontal rhythm consistent across components. Reach for a token rather than a raw pixel value.

## Grid

Composa's grid is the spacing ramp itself. Spacing and radius are authored together as one "grid" — the base-4 ladder that governs gaps, padding, and corner geometry. Layout rhythm comes from applying that grid to flex gaps and padding across components. This is a desktop-app grid: there are no **page-layout** column/container tokens and no responsive breakpoints; the base-4 ramp is the shared rhythm.

It does, though, have a **property-panel column grid** for inspector controls — see below.

## Relational spacing

{/* Inferred from the base-4 keystone (8px) and verified visually against Figma Slides
    template frames: Default node 2723:238794, Design mode node 2723:239738,
    Animate panel node 2723:240665 (file 4kilp0ShQiYsoUPJdleqEH).
    Component evidence: dropdown specimen rows 16px apart; toast label↔button-rail = 8px;
    input↔adjacent-button = 8px; menu parent↔child = 4px; in-row element gaps = space-1/2.
    Carbon vocabulary (Stack / Inset / Inline) borrowed for naming only. */}

The ramp answers "which values exist"; this answers "how much space goes where." Three roles, borrowing Carbon's vocabulary:

- **Inset** — padding _inside_ a container or control.
- **Inline** — horizontal gap between related items on one row.
- **Stack** — vertical gap between elements stacked top-to-bottom.

| Role | Context | Token | px |
|---|---|---|---|
| Inset | Inside a dense 24px control | `--composa-space-1` | 4 |
| Inset | **Default** — control / row padding | `--composa-space-2` | 8 |
| Inset | Panel, card, dialog body | `--composa-space-3` | 16 |
| Inline | Icon ↔ label, within a control | `--composa-space-1` | 4 |
| Inline | **Default** — between related row items | `--composa-space-2` | 8 |
| Inline | Field ↔ its adjacent button | `--composa-space-2` | 8 |
| Stack | Tightly related (list rows, group options) | `--composa-space-1`–`2` | 4–8 |
| Stack | Distinct sibling controls / field rows | `--composa-space-3` | 16 |
| Stack | Group ↔ next group, section breaks | `--composa-space-4`–`5` | 24–32 |

**Principle (from Carbon):** proximity signals relationship. Tighten the step to bind elements together; widen it to separate groups and lift one element's hierarchy. When unsure, default to **8px** and step **up** the ramp to separate — never invent an off-ramp value.

_(proposed)_ Inside a property/inspector panel, controls snap to a **flexible column grid** of **one, two, or three columns** — this is how paired controls (e.g. X/Y, W/H) line up across a panel. It is an inspector layout aid, not a page-layout grid: there are still no responsive breakpoints. Columns share the standard **8px** inline gutter unless a denser pairing calls for 4px.
