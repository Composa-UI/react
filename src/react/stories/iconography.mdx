# Iconography

Icons are line-based glyphs drawn on a consistent frame, sized so they sit cleanly inside dense controls and layout rows.

## Purpose

Iconography governs the size, stroke, and alignment of every glyph in the system so icons read consistently across controls, toolbars, and dense lists. The geometry below is the contract; sourcing rules keep new icons matching it.

## Sizes & weights

The bulk of icons sit in a **24px frame**, drawn with **1px strokes** and **rounded caps (stroke endpoints)**. Each 24px icon can optionally come in a **"small"** variant geared toward a **12px bounding box** for tighter placements. In small, dense contexts — layer lists and similar — icons step down to a **16px frame**.

| Size | Token | Use |
|---|---|---|
| Icon frame | `control-size-icon` (24px) | Default frame for the bulk of icons |
| Glyph | `control-size-icon-glyph` (16px) | Dense contexts (layer lists, properties) |
| Small | 12px bounding box | Optional tighter variant of a 24px icon |

| Property | Value |
|---|---|
| Stroke width | 1px |
| Stroke caps | Rounded |

**Stroke-endpoint icons** are a special case used for stroke/line controls: they share a **16px height**, but their **widths are intentionally irregular** so each fits the input field it labels in the properties panel.

#### Sizing guides

Sizing guidance is provided for common shapes — circles, squares, and rectangles — as alignment templates a glyph should fit within. Treat them as guidelines: use your judgement for more exotic shapes. The guides come in three geometries:

## Usage

Composa does not ship an icon asset library. Components accept icons as **slots** — pass a component type or an already-rendered node from any icon library, and the glyph inherits the slot's box (16px) and color with no extra wiring.

For content icons, source from **Material Symbols** (rounded), which match the family of the built-in structural glyphs (chevrons, check, close). The self-hosted `@composa-ui/icons` Material Symbols runtime is recommended so fill, weight, grade, and optical size stay adjustable; the current runtime default is rounded weight 200. String names resolve to a built-in glyph for backward compatibility.

<Canvas of={IconSlotsStories.Slots} />

- **Do** keep new glyphs on the 24px frame with 1px strokes and rounded caps so they sit consistently with the rest of the set.
- **Do** step down to the 16px glyph size in dense contexts like layer lists.
- **Don't** mix icon families — pull content icons from Material Symbols (rounded) to match the built-in structural glyphs.
- **Don't** hand-size stroke-endpoint icons to a square box; they keep a 16px height with irregular widths to fit their input fields.
