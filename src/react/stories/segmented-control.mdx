# SegmentedControl

A segmented control is a row of mutually exclusive segments on a shared track, with **one
selected at a time**. Use it to switch between a small set of independent states — views or
modes — inside a single surface. (Formerly known as the "Option Strip".)

### Overview

A segmented control presents a small set of independent states and lets the user pick one.
It typically sizes to fit the wide/narrow grid system, occupying **88px** (narrow, 2 tabs)
or **168px** (wide, 3+ tabs) of width depending on the panel.

When a "default" state is needed — meaning *no modification has been made* — it's
represented by a horizontal bar (the leading `minus` segment).

#### When to use

- **SegmentedControl** — to switch between 2 to 6 mutually exclusive options that fit on a
  single row inside one surface (e.g. layout flow, text alignment, number/text transforms).
- **Tabs instead** — for page-level content regions, not an inline mode.
- **Dropdown instead** — when there are too many options to fit on one row.

#### Variants

- **Icon** — each segment is a 24px icon glyph. The common case: layout flow, text
  alignment, number transforms, text transforms.
- **Label** — each segment is a text label, sized to fit its parent container. Used when
  segments read more clearly as words than as glyphs.

### Anatomy

<Canvas of={SegmentedControlStories.Anatomy} />

- **Track** — the shared container (radius `radius-medium`, `overflow-clip`) sitting on
  `color.bg.secondary`.
- **Segment / tab** — a flex `1 0 0` cell, height 24, min-width 1, centered content, radius
  `radius-medium`. The **selected** segment is a raised pill: `color.bg` fill + a 1px
  `color.border` border. Unselected segments sit flat on the track.
- **Icon** — a 24px icon box (icon variant).
- **Label** — `body.medium` text, `overflow-hidden` with `text-ellipsis` for truncation
  (label variant).

There is no divider between segments; separation comes from the raised pill and the track.

### Content

- **Icon variant** glyphs (24px set) cover layout, alignment, and type transforms — for
  example `layout-vertical` / `layout-horizontal` / `layout-wrap`, `align-left` /
  `align-center` / `align-right` / `align-justified`, and the type-transform glyphs
  (proportional/monospace upper- and lowercase numbers, small caps, title case, uppercase).
  The leading `minus` (horizontal bar) is the "default / no modification" segment.
- **Label variant** — size segments to fit their parent container. **If a label is too big
  for the container, it truncates** with a trailing ellipsis.

### Behaviors

Each individual segment is focusable, allowing refined, precise keyboard interaction. The
control is treated like a **list where each item contains a button**, and directional keys
mirror the native browser behavior for such a list.

- Pressing **Enter** or **Space** on a deselected segment **selects** it (the default button
  behavior).
- If the focused segment is **already selected**, Enter / Space does **not** deselect it.
- A **disabled** segment is a rare case — for example, a font that doesn't support small caps
  in the text properties panel. It takes on the disabled color tokens and is supplemented
  with a detailed tooltip.

#### States

| State | Track / segment styling |
|---|---|
| Default — selected | Selected segment: `color.bg` fill + 1px `color.border`. Other segments flat on `color.bg.secondary`. |
| Default — unselected | `color.bg.secondary`, no border. |
| Focus | Focused selected segment: `color.bg` fill + 1px `color.border.selected`. Focused unselected segment: `color.bg.hover` fill, no border. |
| Disabled | Track background: `color.bg.disabled`; label and icon text: `color.text.disabled`. Paired with a tooltip. |

#### Structure

| Element | Size |
|---|---|
| Tab height | 24 |
| Item height (inner pill) | 22 |
| Icon box | 24 × 24 |
| Tab flex | `1 0 0`, min-width 1, centered |
| Strip width — narrow (2-tab) | 88 |
| Strip width — wide (3+ tab) | 168 |
| Strip width — label variant | 240 |
| Strip / tab radius | `radius-medium` |
| Focus tab inner gap | `spacer-1` |
| Focus tab horizontal padding | `spacer-2` |
| Label segment padding | 8 / 4 |

#### Keyboard

| Key | Result |
|---|---|
| `Tab` | Moves focus into the control; behaves just like a list of focusable elements |
| Arrow keys | Move focus between segments, mirroring native browser list behavior |
| `Space` / `Enter` | Selects the focused segment (no effect if it is already selected) |

#### Screen reader & labeling

- The control is treated like a **list where each item contains a button**, so each segment
  is an individually focusable button.
- Give the control a meaningful `label` so the option set is named to assistive tech.
- A disabled segment exposes the disabled state and is supplemented with a detailed tooltip
  explaining why the option is unavailable.

#### Focus

- A focused **selected** segment shows a `color.border.selected` ring — distinct from the
  plain `color.border` of a non-focused selected segment — meeting the visible-focus
  requirement.
- A focused **unselected** segment shows a `color.bg.hover` fill.
