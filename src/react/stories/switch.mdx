# Switch

A switch turns a single setting **on or off**, like a hardware toggle. Flipping it takes
effect immediately — there's no separate confirm step.

<DocTabs>

<DocTab label="Usage">

<Canvas of={SwitchStories.Playground} />

### When to use

- **Switch** — the change applies the moment it's flipped (a preference, a feature toggle).
- **Checkbox instead** — if the change only applies on a later Save/submit, or it's one
  item in a form. _If changing the state doesn't take immediate effect, it's a checkbox._

### Variants

- **On / Off** — the two base states.
- **Mixed** — an indeterminate state for when one switch governs several items that aren't
  all on. Flipping it sets them all to the same state.

### Behavior

- **Immediate effect** — flipping applies the change at once, no save step. This is the
  defining difference from a checkbox.
- **Focus** shows a selection-blue ring; **disabled** dims the control *and* its label.
- A switch usually carries a **label**, and may add a **description** for extra context
  that aligns under the label.
- **Read-only** _(proposed)_ — a non-interactive switch that still takes focus and is
  announced to assistive tech, for review-without-modification contexts. It renders with a
  transparent track outlined in `color.border` and a `color.icon` knob.

### Content _(proposed)_

- The label **names the setting**, not the action — prefer an adjective or noun over a verb
  (e.g. "Grid" / "Grid visible", not "Show grid").
- The label **does not change with the on/off state** — only the control's visual (and any
  state text) reflects on vs off.

_Rows/notes marked (proposed) are inferred, Carbon-benchmarked additions awaiting confirmation._

### Do / Don't

- **Do** pair a switch with a clear label naming the setting.
- **Don't** use a switch for an action that needs confirmation or only applies on submit —
  that's a Checkbox or a Button.

#### Keyboard

| Key | Result |
|---|---|
| `Tab` | Moves focus to the switch |
| `Space` / `Enter` | Toggles the switch on/off |

#### Screen reader & labeling

- The switch exposes a `switch` role with `aria-checked` (`true` / `false`, or `mixed` for
  the indeterminate state).
- **`label` is the accessible name** — not visible text — so always set a meaningful one,
  or the switch is unlabeled to assistive tech.
- A disabled switch disables its label too, so the whole control reads as inactive.

#### Focus

- The focused switch shows a selection-blue ring (`color.border.selected`),
  meeting the visible-focus requirement.
